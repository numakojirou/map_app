# overview — map-app の目的とユースケース

## 何のアプリか

**社内メンバーの現場配置を地図上で共有するための内部向け Web アプリ**。

「誰が、今、どの現場にいるか」という物理的な位置情報を、地図上のマーカーとして可視化することを目的とする。一般公開ではなく社内利用が前提。

## 想定ユーザー

会社のメンバー全員（管理者・現場担当者・事務方）。
誰でもブラウザで開いて、どこに誰が配置されているかを把握できる状態を目指す。

## ユースケース

| シーン | やりたいこと |
|---|---|
| 朝の状況確認 | 今日メンバーがどの現場に出ているかを一目で見たい |
| 現場間の調整 | 近くにいる別メンバーを地図から探したい |
| 不在確認 | 誰がオフィスにいるか／誰が外勤かを判別したい |

## スコープ外（少なくとも現時点）

- 一般ユーザー向けの公開地図サービスではない
- ナビゲーション／ルート検索機能は持たない
- 過去の配置履歴・統計分析は対象外（現在地の共有が主目的）

## 現状の到達点

- 地図表示と静的なマーカー描画は動作する（東京駅・有楽町駅・銀座の喫煙所のサンプル 3 件が表示される）
- マーカーは `public/markers.json` を編集することで増減できる
- Firebase Hosting にデプロイ可能な状態

→ 「会社メンバーの配置共有」という本来用途に向けては、**マーカーをメンバー単位で扱う／更新フローを整える** のが次の山場。詳細は `roadmap.md` 参照。
