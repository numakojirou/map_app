# roadmap — 現状の制約と今後の拡張候補

「社内メンバーの現場配置を地図で共有する」という本来用途に対して、現状はまだ **静的なサンプル地点を表示するだけ** の段階。実用化に向けて拡張すべき領域を整理する。

## 現状の主な制約

| # | 制約 | 影響 |
|---|---|---|
| C1 | マーカーが静的 JSON | 配置を変えるたびに JSON 編集 + 再デプロイが必要 |
| C2 | 認証がない | URL を知っていれば誰でも見られる |
| C5 | 検索／フィルタが無い | メンバー数や現場数が増えると探しづらい |
| C6 | テストが実質ゼロ | リファクタの安全網が無い（`App.test.js` は CRA 雛形のまま落ちる） |
| C7 | UI が地図のみ | 一覧／凡例・カテゴリ表示などの補助情報が無い |

> ※ 旧 C3（メンバーと地点の区別）と C4（更新時刻なし）は Phase 2 で解消済み。

## 技術的に検討が要るポイント

- **Firestore を採用すると `markers.json` のフェッチは廃止** → `App.js` のデータ取得層を抽象化しておくと差し替えが楽
- **react-leaflet 5 + React 19 の相性** — 動作はしているが、StrictMode 二重描画で Leaflet が文句を言うケースが将来出るかもしれない（要観察）
- **マーカー数が増えた場合のパフォーマンス** — 100 件超なら `react-leaflet-cluster` などのクラスタリング検討
